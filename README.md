# 驿窗开源公约



驿窗开源公约数字条款(20210312)
================================





1. 驿窗开源公约简述
================================

驿窗开源公约是专门适用于驿窗项目作品的开源协议(许可证)，其主要目的是为了促进开源共享项目的 **可持续发展**。


驿窗开源公约分两部分，第一部分是把公约的各项权利说明按“极简”的原则逐条公布在网络上，让大家可以随时查看；第二部分是从第一部分公布的条目中挑选一些组合在一起，形成一个最终采纳的具体协议。其中：

  第一部分是全部的权利明细列表，列表会随时间动态变化，主要是会持续增加条目数量，以适应不同作品和不同著作权人的个性化需求。

  第二部分由使用协议的著作权人自行决定，即从第一部分公布的列表中挑选一部分来进行组合，这些组合权利相叠加形成最终采纳的协议条款。

##2. 驿窗开源公约的第一部分 权利明细列表(6000&7000)
================================================================

驿窗开源公约的第一部分类似下面这样：

**6000系列：

  6001. 本作品对个人使用免费。
  6002. 本作品对企业使用免费。
  6003. 本作品允许整体转载或分发。
  6004. 本作品允许通过QQ、微信、钉钉、网络论坛等交流工具进行局部页面转载或分发。
  6005. 本作品允许单独销售。
  6006. 本作品允许捆绑销售。  
  6007. 本作品的著作权所有人不受本协议限制。
  6008. 本作品提供有限担保。
  6009. 本作品有版权回收机制。
  ......
  6999. 本作品对使用者无任何权利限制。(此为演示条目，不开放使用)

*( 6000系列最小6000，最大6999；其中6000保留，暂时不开放使用 )*

**7000系列：

  7001. 本作品对个人使用不免费。
  7002. 本作品对企业使用不免费。
  7003. 本作品禁止整体转载或分发。
  7004. 本作品禁止通过QQ、微信、钉钉、网络论坛等交流工具进行局部页面转载或分发。
  7005. 本作品禁止单独销售。
  7006. 本作品禁止捆绑销售。  
  7007. 本作品的著作权所有人受本协议限制。
  7008. 本作品不提供任何担保。
  7009. 本作品无版权回收机制。
  ......
  7999. 本作品限制以上全部权利。(此为演示条目，不开放使用)

*7000系列最小7000，最大7999；其中7000保留，暂时不开放使用*

其中：

  6000系列为**允许**权益明细，7000系列为**禁止**权益明细；并且，6000系列与7000系列一一对应，  例如：
  
  **6003**为**允许**转载，则对应的**7003**为**禁止**转载；
  **6005**为**允许**单独销售，则对应的**7005**为**禁止**单独销售；

所以，理论上，6000系列中每增加一个条目，7000系列也应该增加一个对应条目，以使两个系列中的条目能够一一对应，以此类推。

所有条目遵循“极简”原则，只描述最小范围的权利，这样可以方便著作权人进行精确组合。

条目在创建时可以对条目添加更详细的解释说明，例如：

  **6007. 本作品的著作权所有人不受本协议限制。**
  
  这一条权利条目内容的详细解释为，最终协议的组合中如果包括这一条，那么所有7000系列的限制都对著作权所有人无效。比如下面的301条款，组合中有7003、7005、7006，不允许销售，但因为301条款中存在6007条目，所以，著作权所有人可以销售。
  
  在这种情况下，著作权所有人可以通过出版发行其作品来获得收入，而其它个人或组织则被限制为不可以出版发行其作品，保障了著作权所有人的最大权益。

##3. 驿窗开源公约的第二部分 权利条目组合(300&500)
================================================================

驿窗开源公约的第二部分类似下面这样：

300. 使用全部6000系列组合。
301. 使用6001、6002、6004、6007；7003、7005、7006、7008组合。
302. 使用6001、6003、6004组合。
...

501. 使用7001、6002、6003组合。
599. 使用全部7000系列组合。
...

驿窗开源公约目前可以使用的组合只有300系列和500系列，其中：

  300系列为个人使用/企业使用均免费；
  500系列为个人使用或企业使用不免费；

著作权所有人从第一部分中挑选好条目后，组合在一起成为300系列或500系列的最终协议条款。

例如，驿窗网站的作品会从6000系列中挑选6001、6002、6004、6007四个条目，并从7000系列中挑选7003、7005、7006、7008四个条目，总计八个条目形成最终的协议条款，这个最终的协议条款被预先定义为301条款。所以，所有使用且仅使用这八个条目的协议都可以称为“驿窗301条款”。(暂命名)

一部作品如果使用了“驿窗301条款”，建议在声明著作权协议时，注明“驿窗301条款”字样；而具体的明细条目，可以同时注明，也可以仅提供一个网络链接，由用户自行到链接处查询。此链接即驿窗已经在网络上公布的第一部分。

##4. 权利组合后的预定义
================================
**已经预定义**


301条款被预定义为6001、6002、6004、6007；7003、7005、7006、7008八个条目的组合，所以，如果著作权/版权所有人使用这八个条目的组合，那么可以直接简化协议名称为“驿窗301条款”，因为301条款已经预定义，可以直接找到明细内容。

预定义项目会随时间逐步扩展，即后续频繁使用的组合会按使用率计算，使用率高的组合会分配预定义300系列条款或500系列条款。这样操作后，可以让高频使用的组合更容易使用。

高频使用的组合需要什么条件才可以升级为预定义的300或500条款，目前仍在拟定中。

**未预定义**

未能预定义的组合，需要著作权所有人逐条说明所引用的条目编号，比如：

  某作品采用驿窗开源公约的6031、6202、6705；7033、7220、7608共六个条目。条目的具体说明可以同时提供，也可以仅提供一个链接，由用户自行到链接处查询。此链接即驿窗已经在网络上公布的第一部分。
  
  此方式要求格式上先写6000系列，再写7000系列；且同一系列按数字由小到大排序。禁止乱序。

著作权所有人可以要求驿窗直接对某个组合进行预定义，但有一个条件，就是此要求需要通过协议维护成员的全票通过。

已经预定义的300或500系列协议，会随第一部分的6000和7000明细条目同时公布，方便著作权所有人直接使用。


##5. 301条款的进一步说明
================================
**1. 301条款中规定了企业可以免费使用(6002)，但不允许销售(7005、7006)，二者似乎有些矛盾，这里解释一下。

  如果企业仅仅是在驿窗网站上(www.ycproject.cn)阅读《GIMP操作指南》，那么完全不违反驿窗301条款的规定。不过，企业不可以打印《GIMP操作指南》，因为打印行为等同于转载或分发，违反301中的7003条目。

  如果一个培训机构使用驿窗网站的《GIMP操作指南》办收费培训班，那么，该机构不可以把《GIMP操作指南》印刷出来给学员，因为此行为等同于分发、销售或捆绑销售，违反301中的7003、7005、7006三个条目。

  同理，因为301条款中禁止转载或分发(7003)，所以，培训机构也不可以为学员提供离线的电子版，也不可以放到培训机构自己的网站上供学员阅读；即，培训机构只可以告知学员驿窗网站的《GIMP操作指南》链接，由学员自行到驿窗网站阅读。

  培训机构给学员提供驿窗网站的《GIMP操作指南》链接时，应当明确指出此《GIMP操作指南》的著作权所有人与该培训机构无任何关联，以防止误导学员，导致学员误认为驿窗网站的《GIMP操作指南》是培训机构专门给学员准备的资料，从而误导学员错误理解培训费用包含的项目。

  如果培训机构在培训实施过程中未尽到说明义务而导致学员误以为培训费用中包含《GIMP操作指南》的资料费，那么就是明显的违反301条款。因为301协议已经明确声明，对个人使用免费(6001)及对企业使用免费(6002)。

  所以，如果培训机构给学员提供驿窗网站的《GIMP操作指南》链接时，明确说明此《GIMP操作指南》的著作权所有人与培训机构无关，学员阅读驿窗网站的《GIMP操作指南》完全不需要付任何费用，那么，培训机构的操作方式就完全符合301条款的全部要求。



**2. 301条款中6004的意义是为了促进交流。

  原则上，301条款禁止转载或分发；但如果无法通过QQ、微信、钉钉、网络论坛等交流工具发作品截图的话，会影响作品本身的推广，也给参与者的交流带来障碍，这违背了301条款的本意，不是301条款希望的结果。
  
  如果作品本身的目的就是为了让更多人使用，那么就应该最大化的方便读者的使用。所以，对某一具体问题的讨论过程中对301条款作品进行局部页面或内容的转载或分发，实际上是受301条款鼓励和支持的；并且，并不限于上面列出的交流工具，即使是企业或组织内部的交流工具，甚至是私人工具，也是受301条款的鼓励和支持的，因为其转载和分发的页面通常都是整部作品的局部一个或几个页面，交流讨论的问题也是非常具体的某些特定的细节问题，对于《GIMP操作指南》来说，这对于推广GIMP使用无疑是非常正面的，所以301条款支持这样做，任何人完全可以通过任何交流工具对受301条款约束的作品的局部进行转载或分发，最典型的就是截图，或者拷贝作品的部分文字与图片。

  不过，如果只是为了方便，而把《GIMP操作指南》的大量页面直接转载到企业或组织内部的网络空间，甚至整体转载，以供企业或组织内部使用，这就不符合301条款的本意，因为这样的操作违反了301条款中关于著作权人权益最大化的目的，分散了著作权人的流量，影响项目后续的可持续性发展。
  
  作品的创作者已经无偿贡献出了作品，仅保留了非常有限的权利；而流量则是一条相对非常重要的权利，301条款支持内容创作者在一定条件下最大化地保留流量权益。
  
  301条款努力尝试为作品的创作者创造有利条件，用来保障创作者的权利在有限条件下尽可能最大化，这样，创作者就有更多机会获得收益，从而继续在创作上进行投入，发布更多的免费作品，最后使更多人因其更多的作品而受益。

  作品的创作者有条件和环境能够持续性地输出作品，作品的受众就能够持续性地免费使用作品，二者相辅相成才能保证开源共享项目的可持续性发展，免费的作品也越来越多，最终形成开源共享项目的良性循环。



综上所述，301条款实际上是为了达到以下目的：

  1. 保障所有通过 www.ycproject.cn 网站阅读作品的人，都是且必须是免费阅读，任何个人或企业、组织都不可以收费；
  2. 保障作品的著作权所有人有机会独家出版发行其作品来获取收益，而其它任何人或企业、组织不经著作权所有人许可都不可以出版发行该作品。此属性保障了著作权所有人的潜在的可能收益。
  3. 任何个人或企业、组织不可以为获取流量而转载/分发受301条款限制的作品。此属性保障了著作权所有人的流量独占特性。
  


##6. 协议冲突
================================
目前6000系列为 **允许** 权利，7000系列为 **禁止** 权利。

如果某个组合在实际应用中出现允许与禁止相矛盾的情况，则以6000系列优先的原则处理。即，优先保护作品使用人的权利，而不是优先保护作品的著作权所有人的权利。

之所以考虑冲突问题，主要是因为6000系列和7000系列的具体条目在制定时可能无法精确预料未来的情况，并且会不断扩充条目数量，在这个过程中有可能并不严谨，导致在实际应用中出现问题。

冲突出现时，最佳解决方法是通知著作权所有人和协议维护成员，尽快升级协议相关文本，对冲突条目重新进行设定和说明。



##7. 驿窗开源公约公布地址
================================

因为公布的内容会被后续查看并挑选使用，所以，需要确保公布位置有一定保障，方便大众查阅，暂时确定在下面的位置公布：

  1. 驿窗项目网站：www.ycproject.cn/license/license.html
  2. 驿窗微信订阅号： 名称：驿窗；微信号：ycproject
  3. 驿窗新浪微博：weibo.com 搜索 驿窗
  4. github(6000&7000系列具体条目的增加，任何人都可以提出需求)

其中，微信订阅号和新浪微博公布的内容 **无法修改**，这有利于协议双方进行线上查询和确认。

*目前协议除了本站之外没啥人用，所以只在 www.ycproject.cn上发布，后续会择机在其它位置公布*


##8. 协议变更
================================

300、500系列预定义条款及6000&7000系列明细条目内容发生变更时，包括增加新的明细条目和删除现有明细条，也包括对现有明细内容的修改。

无论是明细的增加和删除，还是明细内容的局部修改，都会以日期为版本号进行标注；例如版本号为20210305。(需要协议维护成员的全票通过)


6000、7000系列条目的增加，任何人都可以提出要求，因为大家的需求多种多样，而尽量满足多数人的需求本就是协议的设计目的。


##9. 协议的使用与维护
================================
驿窗希望协议的维护在未来能够交由第三方中立机构进行维护。(移交需要协议维护成员的全票通过)

任何人及任何企业、组织可以无条件使用驿窗开源公约，仅要求不违反国家法律。
